Pipeline Overview

1

Qwen3-VL Texture Analysis

Each image is paired with its colored segmentation mask and sent to Qwen3-VL-8B. The model identifies adjacent texture transitions and outputs JSON with 5–9 word discriminative descriptions and mask color assignments for each texture region.

2

Binary Mask Extraction

RGB color matching extracts binary masks from the segmentation. Connected-component filtering ensures only physically adjacent regions are paired. Duplicate transitions are removed by mask IoU and text similarity.

3

OBB-Based Boundary Crop Extraction

Boundary contours between textures are discovered by mask dilation and intersection. Curved boundaries are split into straight segments via polygon approximation. Oriented bounding boxes (OBBs) are fitted to each segment, then dynamically expanded as axis-aligned crops with center-biased boundaries. NMS and visual-similarity deduplication remove overlapping crops.

4

Selective Super-Resolution

Only small crops (<200px min side) are upscaled 2× using Real-ESRGAN x2plus with SDF mask smoothing. Large crops are kept at native resolution to avoid SR artifacts. A unified “final” directory stores the best version of each crop.
